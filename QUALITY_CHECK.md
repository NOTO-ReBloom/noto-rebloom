# 品質確認記録

## 表示・デザイン
- PC幅とスマートフォン幅で全5ページをレンダリングして確認
- 画像に固定ピクセル高を与えず、原則 `height:auto` で元の縦横比を維持
- 横長の農地写真、縦長の土壌写真、チーム写真を用途に合わせて別レイアウトで使用
- 文字の重なり、カードのはみ出し、意図しない横スクロールがないことを確認
- 全ページの色、見出し、ボタン、ヘッダー、フッターを同一デザインシステムへ統一

## 情報設計
- トップ、課題解説、イベント、法人・団体、花タイプ診断の役割を明確化
- 土地の状況を「実施可能な土地は確保済み。複数候補から最適な会場を選定中」に統一
- 法人協賛区分を3万円、5万円、8万5千円で統一
- 参加費500円、見学・応援無料、2026年9月20日を全ページで統一
- 「社内稟議」「社内調整」など相手企業内部を決めつける表現を使用していないことを確認

## データ
- 全国の荒廃農地データは農林水産省の令和6年度資料へ更新
- 能登の農地・農業用施設被害、豪雨被害、営農再開面積は石川県公表資料に基づき整理
- 出典リンクを `learn.html` 内へ明記
- READYFORの表示値に確認日を併記

## 技術検査
- HTML内のローカル参照切れなし
- HTML内の重複IDなし
- `site.js` と `diagnosis.js` の構文検査済み
- モバイルメニュー、クイズ、ランダムアクション、花を咲かせる操作を確認
- 花タイプ診断を56問完了し、結果表示まで確認
- 参加フォームと法人・団体フォームのURLを確認
- ZIP展開試験・ファイル破損検査を実施

## 公開後に利用者側で確認する項目
GitHub Pagesのキャッシュや公開環境はローカル確認と異なる場合があるため、公開後に主要5ページと外部フォームを1回ずつ開いてください。
