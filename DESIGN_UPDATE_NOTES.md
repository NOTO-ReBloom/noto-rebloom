# 今回の主な再構成

## 1. 写真を「背景素材」ではなく「現地の証拠」として扱う
以前のように一定の高さへ押し込まず、元画像の縦横比を保つ額装型レイアウトに変更しました。全景写真、地面の詳細写真、チーム写真を、それぞれ意味の異なる資料として配置しています。

## 2. 課題理解を独立した体験にする
`learn.html` では、用語、全国データ、発生要因、放置による影響、能登の災害被害、プロジェクトができること・できないことを順番に理解できます。数字だけを並べず、「なぜ自分たちが小さく始めるのか」までつなげています。

## 3. イベントを詳しく、しかし誇張せずに伝える
競技、レンゲカップ、安全準備、当日の流れ、持ち物、雨天判断を独立して説明しました。確定事項と調整中事項を分け、参加者が安心して判断できる構成です。

## 4. 法人・団体ページを営業資料ではなく共同提案の入口にする
資金だけでなく、物品、技術・作業、広報、紹介を同じ重さで提示しました。提供できる掲載・報告内容は具体化しつつ、集客や売上など保証できない効果は記載していません。

## 5. 学生プロジェクトらしい遊び心
クイズ、ランダムな「今日からできる一歩」、花を咲かせる操作、花タイプ診断を残し、問題理解の緊張感と参加の楽しさを両立させています。
